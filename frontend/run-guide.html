Step 1

Before you start

  • Read `AGENT.md` and `docs/workflows/MAIN_WORKFLOW.md` before triggering the workflow.
  • If you are using an agent interface, trigger the run with start-agent. Accepted aliases and equivalent start phrases: start, start agent, start work, start-work, run-agent, create-blog, init-blog-agent, agent-start, begin-task, generate-blog, execute-agent.
  • Legacy prompts such as Read AGENT.md and Start AGENT.md remain valid.
  • Review `.env.example` and database settings under `config/DATABASE.md` if you need environment-specific setup.
  • Use `config/NEWS_SOURCES.md` and `config/SCHEDULER.md` if you are adjusting sources or scheduling behavior.

Step 2

Main run commands

Use case Command
Start through PowerShell launcher powershell -ExecutionPolicy Bypass -File .\scripts\start_agent.ps1
Start directly through Python python scripts/run_workflow.py --recency-hours-24
Run continuously with cooldown COOLDOWN_SECONDS=120 bash scripts/run_agent_loop.sh <your-command>
Backfill a missing blog file python3 scripts/blog_file_manager.py --blog-id 5

Step 3

Dashboard and monitoring

  • Serve the live dashboard locally with python3 scripts/agent_dashboard.py --serve --port 8765.
  • Generate a static dashboard snapshot with python3 scripts/agent_dashboard.py --output frontend/dashboard.html.
  • Use `logs/RUN_LOG.md` and `logs/MEMORY_STORE.md` as repository-visible audit trails.

Step 4

What to open after startup

Live UI

Open the served dashboard if you launched `scripts/agent_dashboard.py --serve`.

Static fallback

Open `frontend/dashboard.html` if you generated the snapshot instead of serving.

Workflow view

Use `frontend/blog-workflow.html` for presentation and process explanation.

Operational proof

Check `logs/RUN_LOG.md` for run history and execution evidence.
